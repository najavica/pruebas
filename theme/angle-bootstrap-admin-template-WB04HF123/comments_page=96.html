Hi, thank you for pointing us this issue.
Please use the following LESS code to fix this. Add it at the end of file master/less/app/uiselect.less

.ui-select-multiple.ui-select-bootstrap {
padding: @padding-base-vertical @padding-base-horizontal !important;
min-height: @input-height-base !important;
input {
padding: 0 !important;
margin: 0 !important;
width: auto !important;
height: auto !important;
.placeholder(@text-muted);
}
&.ng-not-empty {
padding-top: @padding-base-vertical - 2 !important;
padding-bottom: @padding-base-vertical - 2 !important;
}
}

Please note that ui-select multiple is a bit tricky to style since the input doesn't really cover the entire area, the plugin uses a wrapper and the input is just to detect focus on element. The styling looks pretty much like a form-control but has some minor difference due to the fact the Tags has margins that consumes space and we haven't found a way to detect when to add/remove margins in order to detect when the amount of tags force to the input to break the first line..

Hi, there's a problem with whirl package in package.json file, not sure what happened but it started to fail recently so we decide to fix this by using our own repo until found a solution.
To solve this please edit package.json and the remove the whirl package and replace with the following:

"whirl": "themicon/whirl",

Then run commit changes and run again the installation.

There's also another issue with the gem used for PosgreSQL, it was updated to 1.0 recently and it seems to be not working fine on rails (https://github.com/rails/rails/issues/31673)
To solve this edit your Gemfile and replace 'gem pg' with the following

gem 'pg', '~> 0.20'

Our apologies for so many issues. Just in case, following the steps detailed in documentation works fine. Only problem seems to be the packages mentioned that we will fix in the next update.

About Bootstrap

Bootstrap is designed to help people of all skill levels – designer or developer, huge nerd or early beginner. Use it as a complete kit or use it to start something more complex.

Unlike other front-end toolkits, Bootstrap was designed first and foremost as a styleguide to document not only its features, but best practices and living, coded examples.

Built to support new HTML5 elements and syntax with progressively enhanced components.

Browser support

Bootstrap is tested and supported in major modern browsers like Chrome, Firefox, and Internet Explorer.

  • Latest Chrome
  • Latest Firefox
  • Latest Safari
  • Latest Edge
  • Internet Explorer 10/11
  • Latest Opera

Features & includes

Bootstrap provides an unparalleled array of features and reusable components that let you hit the ground running:

  • Responsive 12-column grid
  • 12 custom, modular jQuery plugins
  • CSS styles for forms, navigation & more
  • Dozens of reusable components
  • Components are scaled according to a range of resolutions and devices
  • Built with Sass (CSS included)
  • Complete styleguide documentation
