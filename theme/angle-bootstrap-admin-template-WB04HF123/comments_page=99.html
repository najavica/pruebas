themicon SELLER
1 month ago
Only possible workaround for this found at the moment is by compiling styles with cli tools instead of processing sass and rtl into the app.
If you want to try it, please follow this steps

1- Rollback all previous changes provided to add rtlcss in app.component.ts and app.component.scss
2- Add the following scripts to package.json

"build-rtl-sass": "npm run build-sass && npm run build-rtl",
"build-rtl": "npm run build-app-rtl && npm run build-bs-rtl",
"build-sass": "npm run build-app-sass && npm run build-bs-sass",
"build-app-rtl": "rtlcss src/app/shared/styles/app.css",
"build-bs-rtl": "rtlcss src/app/shared/styles/bootstrap.css",
"build-app-sass": "node-sass src/app/shared/styles/app.scss -o src/app/shared/styles/",
"build-bs-sass": "node-sass src/app/shared/styles/bootstrap.scss -o src/app/shared/styles/"

3- Run "npm run build-rtl-sass"
This command will generate css stylescheet in LTR and RTL, like this

shared/styles/app.css
shared/styles/bootstrap.css
shared/styles/app.rtl.css
shared/styles/bootstrap.rtl.css

4- Now you can @import the following in src/style.scss

//== Bootstrap
@import "./app/shared/styles/bootstrap.rtl.css";
//== Application
@import "./app/shared/styles/app.rtl.css";

Please consider that paths to node_modules (url(), etc) are relative to src/app/shared/styles/ folder. For example, url('../../../../node_modules/...')
The tilde (~) seems to be not working with this approach.

Regards
