jkildedal PURCHASED
3 years ago
Hi Damian

Thanks for your answer.

I think you misunderstood my question, though :)

The conventions you use to define and pass your modules, as described in the document you showed me, is very fine. In fact I agree with many of the conventions you use.
But when you pass your predefined functions, there are generally 2 ways to do it, the one being minifyable and the other not.

Example (directives.js):
function iboxTools($timeout) {
return {
restrict: 'A',
scope: true,
templateUrl: 'views/common/ibox_tools.html',
controller: function ($scope, $element) {
// content left out for clarity...
}
};
};

angular
.module('inspinia')
.directive('iboxTools', iboxTools);

This will not minify correctly since the $timeout argument will be renamed by the minifyer and since Angular injects its dependecies by their variable names. In fact this example shows 2 minification problems, since the $scope and $element arguments of the controller property will also be minified.
If instead the directives were defined in the following way, it would be minified correctly:

function iboxTools($timeout) {
return {
restrict: 'A',
scope: true,
templateUrl: 'views/common/ibox_tools.html',
controller: ['$scope', '$element', function ($scope, $element) {
// content left out for clarity...
}]
};
};

angular
.module('inspinia')
.directive('iboxTools', ['$timeout', iboxTools]);

Note here how both the definition of the controller property inside the directive function as well as the passing of the directive function are done differently.

Can you fix these things in your next release?
It should be quite easy to fix :)
