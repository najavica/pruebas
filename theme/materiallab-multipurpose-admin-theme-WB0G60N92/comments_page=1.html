Hi,

Working on wrapping up the docs right now, I'll have it ready within the next couple of days.

Are you loading the dataTables from the full version or the seed version project? If you're loading it from the seed project then you'll need to add the dataTables plugin via Bower. The seed project is a stripped down version of the full project. It's designed to have the bare minimum so you can add only what you need for a starting point.

The full project has all of the plugins installed, so if you're in the full project folder you can take a look at 'app/js/modules/dataTables.js' line 6 is where the dataTable initialisation call is. It's wrapped in a function called 'mwDataTables' on line 1. Scroll to the bottom of that page and you can see that 'mwDataTables' is being exported.

Every .js file that's in the Modules folder will have their methods getting exported or imported. Now just outside of the Modules folder is a file called app.js it's located here app/js/app.js this is the file that all of those JavaScript files that are in the Modules folder are getting imported into. This app.js file will import all of those methods at the top then on line 155 of app.js they are getting loaded through a document ready function in short hand notation $(function() { ... });

Hope this helps!

Best regards,
Jason

authenticgoodsco SELLER
12 months ago
Hi,
Thank you very much! Sorry to hear you're having some problems. It's actually designed to be completely modular so you can use only what you need.

There's 2 ways you can go about customizing it. You can begin with the Full Project and remove what you don't need and then begin to customize. Or you can start with the Seed Project and start adding only what you need. If you're having problems and don't know where to start I would recommend starting with the Seed Project first. This only includes the base functionality for the theme to be styled properly and make it navigable, then you can start working in only the modules you want.

If you not quite sure how to customize it from the Full Project folder, allow me to explain. Have a look at the app folder, this is your working directory where all of the components are separated out. Open up the 'vendor.json' file, there you can remove any of extra plugins that are not needed. Next look at the 'app/js/app.js' file, this file acts as one large include to all of the JavaScript files that's inside of 'app/js/modules/...'. So for any of the JavaScript files you don't need just remove the import reference and where they get called...which all live on the app/js/app.js' file.

Next have a look at the documentation where it walks you through how to create your own custom bundles.

Hope this helps.

About Bootstrap

Bootstrap is designed to help people of all skill levels – designer or developer, huge nerd or early beginner. Use it as a complete kit or use it to start something more complex.

Unlike other front-end toolkits, Bootstrap was designed first and foremost as a styleguide to document not only its features, but best practices and living, coded examples.

Built to support new HTML5 elements and syntax with progressively enhanced components.

Browser support

Bootstrap is tested and supported in major modern browsers like Chrome, Firefox, and Internet Explorer.

  • Latest Chrome
  • Latest Firefox
  • Latest Safari
  • Latest Edge
  • Internet Explorer 10/11
  • Latest Opera

Features & includes

Bootstrap provides an unparalleled array of features and reusable components that let you hit the ground running:

  • Responsive 12-column grid
  • 12 custom, modular jQuery plugins
  • CSS styles for forms, navigation & more
  • Dozens of reusable components
  • Components are scaled according to a range of resolutions and devices
  • Built with Sass (CSS included)
  • Complete styleguide documentation
