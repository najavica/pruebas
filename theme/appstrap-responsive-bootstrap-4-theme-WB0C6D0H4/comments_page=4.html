Hi, what is the best way to override SASS variables in the appStrap theme. For instance, I want to change the color green of $skinGreen but I don't want to edit the downloaded theme files so that I can easily update the theme down the road.

Thanks

Jeremy
themelize_me SELLER
11 months ago
Hi Jeremy,

The simplest way is:

1) to take one of the example colour variations in the /assets/scss/colour-variations & copy, paste & rename it to say custom-colour.scss.
2) Open it up and change the "$_primary_colour" variable to the colour you want
3) Open up terminal and type "gulp" to recompile the sass files, see the README.txt file in the documentation folder for more information on that
4) Go to /assets/css & you should see custom-colour.css & custom-colour.min.css
5) Include either custom-colour.css OR custom-colour.min.css in your pages after the line "<link href="assets/css/theme-style.min.css" rel="stylesheet">"

Let us know any more questions
Thanks
Tom

themelize_me SELLER
11 months ago
Hi,

Thanks for getting in touch.

/assets contains all the css, js, plugins & images the whole theme demo needs to run. If you only need a few pages you could trim the images to just the ones you need by cross-referencing the images used in the HTML code on the pages you need (look for img tags & data-bg-img attributes).

Javascript & plugin files should not be altered as the plugins are only loaded on demand when needed so pages aren't bloated unnecessarily by unused plugins.
CSS only requires the main theme-style.min.css (or theme-style.css) & theme-shop.css if you need the shop theme. Also if you want to disable flexbox support for IE you'll need the no-flexbox.css file.
If you have experience with sass you can create your own custom .css files using only the AppStrap css code you need. We'll be doing some tutorials on this in the coming months.

/node_modules can be removed and is only needed if you are using gulp to compile you sass to css.

Hope that helps, any other questions please email us at [email protected]
Thanks
Tom

Take a look in the README.txt file in the /documentation folder, it says;

10. Pages in subfolders
----------------------------------------------------
If you want to put your pages into subfolders you will need to alter the path to the plugins so the script.js file can find them correctly.
To do this (since AppStrap version 3.0.5) you can simply add the following to the body tag for all pages that are within subfolders:
<body data-plugins-localpath=“LOCAL-PATH”>

If you pages is within 1 subfolder it would look something like:
<body data-plugins-localpath=“../assets/plugins/”>

In your site I think this should work:

<body data-plugins-localpath=“http://test.vaner.eu/assets/plugins/”>

Hi again,

Here are some snippets to help.

1. Find the data-toggle="jpanel-menu" element and within it change the content to:
<i class="fa fa-bars pl-1 menu-icon-off"></i> <i class="fa fa-times pl-1 menu-icon-on"></i> <span class="d-block">Menu</span>

The icons can be whatever icons you like that are included in the theme: http://demos.themelize.me/appstrap3.0.7/theme/elements-icons.htm

2. In your assets/css/custom-style.css file add the following:
[data-toggle="jpanel-menu"] .menu-icon-on {
display: none;
}
[data-toggle="jpanel-menu"].open .menu-icon-on {
display: block;
}
[data-toggle="jpanel-menu"].open .menu-icon-off {
display: none;
}

That should do it
Thanks
Tom

About Bootstrap

Bootstrap is designed to help people of all skill levels – designer or developer, huge nerd or early beginner. Use it as a complete kit or use it to start something more complex.

Unlike other front-end toolkits, Bootstrap was designed first and foremost as a styleguide to document not only its features, but best practices and living, coded examples.

Built to support new HTML5 elements and syntax with progressively enhanced components.

Browser support

Bootstrap is tested and supported in major modern browsers like Chrome, Firefox, and Internet Explorer.

  • Latest Chrome
  • Latest Firefox
  • Latest Safari
  • Latest Edge
  • Internet Explorer 10/11
  • Latest Opera

Features & includes

Bootstrap provides an unparalleled array of features and reusable components that let you hit the ground running:

  • Responsive 12-column grid
  • 12 custom, modular jQuery plugins
  • CSS styles for forms, navigation & more
  • Dozens of reusable components
  • Components are scaled according to a range of resolutions and devices
  • Built with Sass (CSS included)
  • Complete styleguide documentation
