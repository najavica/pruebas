As Heather implies, this theme loads javascript files in a non-optimal fashion which will likely render it incompatible with many plug-ins.

For those who wish to remedy this, I suggest you comment out / delete the script links in footer/scripts.php and add the following to functions.php:

// Register and Enqueue javascript files
function myveryown_load_javascript_files() {
wp_register_script( 'jquery', get_template_directory_uri() . '/js/jquery.min.js',false,'1.11.1', true );
wp_register_script( 'jquery.cookie', get_template_directory_uri() . '/js/jquery.cookie.js', array('jquery'),'1.4.1', true );
wp_register_script( 'bootstrap', get_template_directory_uri().'/js/bootstrap.min.js', array('jquery'), '3.3.4', true );
wp_register_script( 'bootstrap-switch', get_template_directory_uri().'/js/bootstrap-switch.min.js', array('jquery'), '3.2.1', true );
wp_register_script( 'wow', get_template_directory_uri().'/js/wow.min.js', array('jquery'), '1.0.2', true );
wp_register_script( 'slidebars', get_template_directory_uri().'/js/slidebars.js', array('jquery'), '0.9.4', true );
wp_register_script( 'bxslider', get_template_directory_uri().'/js/jquery.bxslider.min.js', array('jquery'), '4.1.2', true );
wp_register_script( 'holder', get_template_directory_uri().'/js/holder.js', array('jquery'), '2.3.2', true );
wp_register_script( 'buttons', get_template_directory_uri().'/js/buttons.js', array('jquery'), true );
wp_register_script( 'mixitup', get_template_directory_uri().'/js/jquery.mixitup.min.js', array('jquery'), '2.1.7', true );
wp_register_script( 'countdown', get_template_directory_uri().'/js/jquery.countdown.min.js', array('jquery'), '2.0.4', true );
wp_register_script( 'circles', get_template_directory_uri().'/js/circles.min.js', array('jquery'), '0.0.4', true );
wp_register_script( 'app', get_template_directory_uri().'/js/app.js', array('jquery','wow','bootstrap-switch'), true );

wp_enqueue_script( 'jquery' );
wp_enqueue_script( 'jquery.cookie' );
wp_enqueue_script( 'bootstrap' );
wp_enqueue_script( 'bootstrap-switch' );
wp_enqueue_script( 'wow' );
wp_enqueue_script( 'slidebars' );
wp_enqueue_script( 'bxslider');
wp_enqueue_script( 'holder' );
wp_enqueue_script( 'buttons' );
wp_enqueue_script( 'mixitup' );
wp_enqueue_script( 'countdown' );
wp_enqueue_script( 'circles' );
wp_enqueue_script( 'app' );
}

// Let's load the javascript files with the wp_enqueue_scripts hook
add_action( 'wp_enqueue_scripts', 'myveryown_load_javascript_files' );

I hope this helps those who've already purchased and were hoping to use plug-ins. You should check the above code against the links in scripts.php to ensure that the versions haven't changed in updates etc.

I had this same issue.

Solution:
You have to install via FTP. I believe this error shows up when you try to install via Wordpress Admin Dashboard > "Upload Theme".

It did for me anyways. After reading the documentation I noticed it said to upload via FTP.

1. Open FTP connection to your Wordpress site.
2. Put the "Artificial-Reason-WP" folder (second one) into your 'wp-content/themes" folder, where twentyfourteen/twentyfifteen/etc. are located
3. Put the plugins in the "wp-content/plugins" folder
4. Once those uploads are complete, when you open your Wordpress Admin Dashboard, go to Appearance > Themes
5. Artificial Reason WP should be there now (might need to click refresh if you already had the Themes page open)
6. Click "Activate"
7. You're good to go

Nevermind, figured it out - I'm a dummy (-_-)

Solution:
1. Copy and paste the code from the Carousel page to whatever page you want it on.
2. On the Carousel page, scroll down to where the Header/Footer plug-in added it's boxes.
3. Copy the entire code from the Footer section, then paste it to the Footer section of whichever page you want your carousel on.
4. Delete the MX1 - MX5 parts, except for the one that corresponds to the Carousel you copied/pasted. (i.e. if you used the second one, save MX2 and delete MX1, 3, 4, and 5.)
5. DO NOT DELETE the first two lines (script type and document ready) or the last two lines (the "});" or the /script).
6. Preview the page to ensure you got it right, then make changes to images/code as you see fit.

You could just re-write it yourself, but this is the way I did it (at least for this time while trying to figure it out). I went through all potential CSS and JS pages looking for any reference to MX1-5, checked the actual MX website, etc., and finally after spending several hours searching and doing the whole trial-and-error process I accidentally noticed the footer part and realized my mistake.

Anyways, hope this saves someone some time.
