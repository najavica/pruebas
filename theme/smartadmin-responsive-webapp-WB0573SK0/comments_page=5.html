Hello again,

I was just playing around with the angular version and realized that there are memory leaks. If you browse around the menu for a while the memory usage keeps climbing and climbing until eventually, for me, it hit 1GB and then the tab crashed in chrome. Other browsers are probably worse.

The worst offender seems to be data tables but that's not the only page that causes the leaks. I'm guessing pageSetUp() introduces some as well because there seems to be a lot of initialization code which happens on each route change but no cleanup.

When writing applications as SPA's with a framework like Angular it becomes extremely critical to handle cleaning up resources, DOM references, event listeners, etc. on each route change.

In a normal stateless application such as your PHP version everything is cleared each time the page is refreshed so memory leaks aren't as big of a deal. When you're in a stateful application where the application lives in the browser's memory indefinitely while the user clicks around the app then memory leaks can cause trouble fast. This is especially true for an admin type application where the user could conceivable browse around views with complex tables, graphs, etc. for long periods of time without doing a full refresh of the page.

Any input on this? Is this something you plan on addressing? Thanks!

Cheers,
Drammy
MyOrange SELLER
3 years ago
Hi Drammy,

We have spent countless nights trying to figure out a way to do this, at the end we realized that AngularJS was not well built for the animation part. The way it works, AJS copies the newly loaded content to another container which sits behind the current content, it then fades out the old content while fading in the new, it executes the javascript functions for the new page before the old content gets removed.

This confuses Jarviswidget as it does not know which page is being loaded, as both page content (though one of then is hidden) are being present at the same time. We have emailed google's AJS team about this and waiting for an update. The AJS's animation engine needs to change slightly for Jarviswidget to work properly.

You can still use both but you will lose the localStorage benefit. To do so, open the app.js file and turn off localstorage for Jarviswidget, you will then be able to use anAnimate and also Jarviswget. You can modify the code further and save/load widget settings via datatabse which might give you more control.

Thanks,
MyOrange

Hello,

For the bootstrap datepicker you can refer to this website here for a detailed documentation of how to change these values:

http://tarruda.github.io/bootstrap-datetimepicker/

The override would be something like:

$.fn.datetimepicker.defaults = {
maskInput: true, // disables the text input mask
pickDate: true, // disables the date picker
pickTime: true, // disables de time picker
pick12HourFormat: false, // enables the 12-hour format time picker
pickSeconds: true, // disables seconds in the time picker
startDate: -Infinity, // set a minimum date
endDate: Infinity // set a maximum date
};
