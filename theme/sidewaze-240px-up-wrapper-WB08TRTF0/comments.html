The footer is part of the layout. So you can't remove it without it affecting everything else. There's a lot of CSS for each breakpoint for each area, if you are familiar with CSS and media queries, then you can go into the CSS and adjust. Remove the html for the footer and then go into the CSS and make the adjustments inside the media queries. The navigation has two links to signin and sign up, if you remove that html, the navigation will move up. You can also see the top padding for that area in the CSS and adjust. Use your web developer tools or open up the html to identify the class used.

Your question indicates that this wrapper may be more advanced for your needs. Most layouts are made horizontally as that is the easiest for less experienced developers to adjust, this is an unusual layout and requires rather good CSS skills to adjust.

Q: What is the purpose / function of the faux-col sections below the footer? Just curious to understand the thinking here.

Cheers,
J.
christina SELLER
2 years ago
Thanks! That's explained in the sidewaze_columns.css file comments. The real columns are floated, the faux columns are fixed, if you only had fixed columns, the site would behave oddly depending on the height. When you have the faux (position:fixed) columns behind the real, floated columns, when the content is shorter in one column and longer in another, the rules follow the user as he or she scrolls. It's been a long time since I created this wrapper, there may be better ways to address this now. I would probably use jquery these days.
