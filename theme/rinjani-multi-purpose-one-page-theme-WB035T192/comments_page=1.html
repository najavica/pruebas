Hi micko,

Thanks for buying a Rinjani template. Sorry for the late reply.

Seems plugins that I use for the detection of mobile browsers do not consider android tablet as a mobile browser.

In the HTML file, search for the "$(window).load", and change the following exact text with this:

$(window).load(function(){
var ua = navigator.userAgent.toLowerCase(),
isAndroid = ua.indexOf("android") > -1,
isIpad = ua.indexOf("ipad") > -1;

if (jQuery.browser.mobile === false && !isAndroid && !isIpad){
// This is for Parallax using Skrollr
var s = skrollr.init({
forceHeight: false
});

// Refresh Skrollr after resizing our sections
s.refresh($('.parallax'));
}
});

Great theme, quick question:

I've been trying to figure out which plugin is controlling the brightness of the video background. Could you enlighten me? Thanks!
etap SELLER
2 years ago
Hi bwrodgers,

Thanks for purchasing Rinjani, and sorry for the late reply.

Option A:
1. Just open your assets/css/rinjani.css file. Find this css code:

#intro:before {

2. You can set the opacity of a background (video) of an overlaying div (#intro) with this code:

// Example 1
#intro:before {
background: rgba(0, 0, 0, 0.2);
}

// Example 2
#intro:before {
background: rgba(0, 0, 0, 0);
}

Option B:
1. First, you must have to download the latest update of the jquery.mb.YTPlayer plugin.
https://github.com/pupunzi/jquery.mb.YTPlayer

2. Open your assets/js/main.js file, under YOUTUBE VIDEO BACKGROUND section, find and change this line:

var ua = navigator.userAgent.toLowerCase(),
isAndroid = ua.indexOf("android") > -1;

var filters = {
brightness: 200 // 1 - 400
};

if(jQuery.browser.mobile === false && !isAndroid){
$('.player').YTPlayer({
videoURL: 'https://www.youtube.com/watch?v=Ufnf0ecwzVI',
containment: 'body',
autoPlay: true,
mute: true,
startAt: 137, // Set the seconds the video should start at.
opacity: 1
}).YTPApplyFilters(filters);
}

We hope you find this helpful. Kindly submit your request through the contact form on our profile page :)

Hi,

You are right, that will be something I would fix in the next update. If you need a quick fix for that, here are the steps, it has been tested and worked.

Please open assets/js/main.js file, under "JQUERY FOR PAGE SCROLLING FEATURE" section, copy this js code:

BEFORE:
var pageScroll = function(){
$('.page-scroll a').bind('click', function(e){
e.preventDefault();

var $anchor = $(this);

var offset = $('body').attr('data-offset');

if($('.navbar.navbar-fixed-top').hasClass('side-menu') && $(window).width() >= 992){
$('body').data('offset', 1);
offset = $('body').data('offset');
}

$('html, body').stop().animate({
scrollTop: $($anchor.attr('href')).offset().top - (offset - 1)
}, 1500, 'easeInOutExpo');
});
};
